Planno is an existing production-style application.

The objective is quality elevation, not reconstruction.

Before modifying code:

- Analyze current implementation.
- Identify technical debt risks.
- Avoid large-scale rewrites.

Refactoring Philosophy:

- Extract logic before behavior modification.
- Add validation coverage where missing.
- Improve structure incrementally.

Do not rewrite working subsystems unless there is a strong reliability or safety justification.

Focus on:
- Security hardening
- Observability
- UX polish
- Validation completeness
